# Калькулятор здоровья (hexlet-vibecoding)

## Описание проекта

Интерактивное веб-приложение для расчета индекса массы тела (BMI) и дневной нормы калорий. Приложение предназначено для образовательных целей и демонстрации работы с HTML, CSS и JavaScript.

## Структура проекта

```
hexlet-vibecoding/
├── index.html          # Главная страница с HTML-структурой
├── styles.css          # Стили и адаптивный дизайн
├── script.js           # Логика калькуляторов и управление UI
├── README.md           # Документация проекта
└── CLAUDE.md           # Этот файл
```

## Архитектура

### Классы JavaScript

#### `BMICalculator`
Отвечает за расчет индекса массы тела.

**Методы:**
- `calculate(heightCm, weightKg)` → число - расчет BMI
- `getCategory(bmi)` → строка - определение категории (underweight, normal, overweight, obese)
- `getCategoryText(category)` → строка - текстовое описание категории на русском
- `getDescription(category)` → строка - рекомендации для категории

**Формула:** BMI = вес(кг) / (рост(м))²

#### `CalorieCalculator`
Отвечает за расчет дневной нормы калорий.

**Методы:**
- `calculateBMR(age, gender, heightCm, weightKg)` → число - расчет базового метаболизма
- `calculateDailyCalories(bmr, activityLevel)` → число - расчет дневной нормы
- `getMacroNutrients(calories)` → объект - распределение макронутриентов

**Формула:** Harris-Benedict formula для расчета BMR, затем умножение на коэффициент активности

#### `UIController`
Управляет пользовательским интерфейсом и координирует работу калькуляторов.

**Функции:**
- Инициализация обработчиков событий
- Переключение вкладок
- Валидация и обработка форм
- Отображение результатов
- Real-time расчеты

## Функциональность

### Вкладка 1: Индекс массы тела (BMI)
- Ввод: рост (см), вес (кг)
- Вывод: значение BMI и категория с рекомендациями
- Категории:
  - Недостаточный вес (BMI < 18.5) - синий
  - Нормальный вес (BMI 18.5-24.9) - зелёный
  - Избыточный вес (BMI 25-29.9) - оранжевый
  - Ожирение (BMI ≥ 30) - красный

### Вкладка 2: Дневная норма калорий
- Входные параметры: возраст, пол, рост, вес, уровень активности
- Вывод: дневная норма калорий (ккал/день)
- Дополнительно: распределение макронутриентов
  - Белки: 25% калорий / 4 ккал/г
  - Жиры: 30% калорий / 9 ккал/г
  - Углеводы: 45% калорий / 4 ккал/г

## Стили и дизайн

### Цветовая схема
- Основной градиент: фиолетовый (#667eea) → розово-фиолетовый (#764ba2)
- Фон: белый (#ffffff)
- Текст: тёмно-серый (#333333)

### Адаптивность
- Desktop: полная ширина контейнера до 600px
- Tablet: оптимизированный layout
- Mobile: одноколонный layout, увеличенные сенсорные элементы

### Key CSS классы
- `.tabs` - контейнер вкладок
- `.tab-button` - кнопка вкладки
- `.tab-content` - содержимое вкладки
- `.calculator-card` - карточка калькулятора
- `.result` - блок результатов
- `.bmi-${category}` - стили результатов BMI по категориям
- `.macro-split` - сетка распределения макронутриентов

## Инструкции по разработке

### Локальный запуск
1. Клонируйте репозиторий
2. Откройте `index.html` в браузере

### Добавление новых функций

#### Добавление нового калькулятора
1. Создайте класс калькулятора, наследуя логику
2. Добавьте новую вкладку в HTML
3. Добавьте методы в UIController
4. Добавьте стили в CSS

#### Изменение формул расчета
- BMI: редактируйте метод `calculate()` в `BMICalculator`
- Калории: редактируйте `calculateBMR()` или коэффициенты активности в HTML

## Соглашения о кодировании

### JavaScript
- Используются ES6+ классы
- Методы описываются в порядке: инициализация, утилиты, отображение
- Комментарии только для сложной логики
- Нейтральные имена переменных (heightCm, weightKg с указанием единиц)

### HTML
- Используются семантические теги
- ID для элементов форм и результатов
- Data-атрибуты для переключения функциональности

### CSS
- BEM-подобная нотация для класов
- Custom properties для цветов (опционально)
- Mobile-first подход в media queries

## Тестирование

### Ручное тестирование
- Проверьте расчеты с известными значениями
- Тестируйте на мобильных браузерах (Chrome, Safari)
- Проверьте валидацию форм (пустые поля, отрицательные значения)

### Граничные значения
- Мин рост: 100 см, макс: 250 см
- Мин вес: 30 кг, макс: 300 кг
- Мин возраст: 10 лет, макс: 100 лет

## Важные замечания

- Приложение предназначено в образовательных целях
- Результаты не являются медицинским советом
- Используется упрощенная формула Harris-Benedict (для демонстрации)
- Все расчеты выполняются на стороне клиента (JavaScript)
